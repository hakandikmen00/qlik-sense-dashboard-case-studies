# Qlik Sense Dashboard Case Studies

Bu repo, **Qlik Sense** ile geliştirilen dashboard ve reporting yaklaşımlarını **case study** formatında sunmak için hazırlanmıştır.  
İçerikler yalnızca görsel ekranlardan değil; **iş ihtiyacı, KPI yapısı, dashboard kurgusu, filtreleme mantığı ve karar destek yaklaşımından** oluşur.

This repository presents **Qlik Sense** dashboard and reporting approaches in a **case study** format.  
The content is not limited to visuals; it also includes **business needs, KPI structures, dashboard logic, filtering approach, and decision-support perspectives**.

<br>

## Repo’nun Amacı / Purpose

Bu repo’nun amacı, Qlik Sense projelerini yalnızca teknik veya görsel çıktılar olarak değil;  
**iş problemi, kullanıcı ihtiyacı, ölçüm yapısı ve analitik yaklaşım** ile birlikte sunmaktır.

Amaç, dashboard tasarımını sadece “ekran üretimi” olarak değil;  
**anlamlı KPI’lar, doğru görünürlük ve karar desteği sağlayan bir yapı** olarak ele almaktır.

The purpose of this repository is to present Qlik Sense projects not only as technical or visual outputs, but as complete analytics solutions shaped by **business problems, user needs, measurement structures, and reporting logic**.

The focus is not only on building screens, but on creating dashboards that provide **meaningful KPIs, visibility, and decision support**.

## Case Studies

### 1) Executive / Management Dashboard
Üst yönetime yönelik KPI özeti, görünürlük ve karar destek yaklaşımını ele alan dashboard kurgusu.  
Dashboard approach focused on executive visibility, KPI summary, and decision support.

### 2) Operational Performance Dashboard
Operasyon ekipleri için günlük ve dönemsel performans takibini destekleyen, aksiyon odaklı görünürlük sağlayan yapı.  
A dashboard structure designed for daily and periodic performance tracking with action-oriented visibility for operations teams.

### 3) Agency / Sales / Production Dashboard
Üretim, satış, kanal veya acente performansını izlemeye yönelik dashboard yaklaşımı ve KPI tasarımı.  
A dashboard and KPI approach built for monitoring production, sales, channel, or agency performance.

<br>

## Yaklaşım / Approach

Bu repodaki case study’ler hazırlanırken yalnızca görsel tasarım değil;  
**iş ihtiyacının anlaşılması, doğru KPI yapısının kurulması, kullanıcı dostu dashboard akışının tasarlanması ve karar desteği sağlayan görünürlüğün oluşturulması** esas alınmıştır.

Her case study şu temel mantıkla ele alınır:

- İş problemi veya ihtiyaç nedir?
- Hedef kullanıcı kimdir?
- Hangi KPI’lar izlenmelidir?
- Dashboard hangi sorulara cevap vermelidir?
- Filtre yapısı ve gezinme mantığı nasıl olmalıdır?
- Sağlanan görünürlük hangi kararları destekler?

The case studies in this repository focus not only on visual design, but also on **understanding business needs, defining the right KPI structure, designing user-friendly dashboard flows, and creating visibility that supports decision-making**.

Each case study is built around the following questions:

- What is the business problem or need?
- Who is the target user?
- Which KPIs should be tracked?
- What questions should the dashboard answer?
- How should the filtering and navigation flow be designed?
- What decisions does the visibility support?
